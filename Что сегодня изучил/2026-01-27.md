Так же сегодня изучал потоки ввода вывода (STDIN,STDOUT,STDERR)
STDIN- 0 (ввод командам и утилитам)
STDOUT - 1 (вывод команд и утилит)
STDERR - 2 (ошибки в работе команд/утилит)

Примеры использования
ls -l > test.txt - перенаправит вывод команды в файл
ls -l >> test.txt - добавит вывод в тот же файл
ls -l 2> error.txt - перенаправит вывод ошибок в файл
ls -l > all.txt 2>&1 - перенаправит вывод ошибок туда же куда и вывод команды, т.е. все будет в одном файле (аналог &>)
| - пайплайн, можно выстраивать цепочки передачи вывода одной команды на вход другой, например:
ls -l | grep "logs"
2>/dev/null перенаправление в никуда(файл который ничего не сохраняет), используем тогда, когда нам не важен вывод например ошибок и мы хотим видеть только вывод команды
sort < unsorted.txt - передаем на ввод команде(STDIN)
<<< - "Here string" передаем обычную строку
<< EOF - можем передать на вход несколько строк(блок завершается EOF)
tee - используется для одновременного вывода на экран и в файл, например:
ls -l | tee file.txt
ls -l | tee -a file.txt - для перезаписи в тот же файл
