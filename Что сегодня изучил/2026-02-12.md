Сегодня изучал пакетные менеджеры в Linux.
Пакетные менеджеры необходимы для быстрой и просто установки пакетов (по,утилит) со всеми необходимыми зависимостями.
Например для Ubuntu/Debian пакетный менеджер apt

sudo apt update - обновляет индексы пакетов, т.е. информацию о пакетах, какие можно обновить и т.д
sudo apt install название пакета - позволяет установить пакет
sudo apt upgrade \[имя пакета] - обновление пакетов
Так же пакеты можно удалять, можно удалять неиспользуемые зависимости и т.д.

По можно устанавливать из исходников, например забирая их из репозиториев гитхаба, например через wget, обычно мы получаем .tar.gz /bz2 архивы, далее их нужно распаковать, например
tar -xzvf archive_name
а после этого компилировать и устанавливать, например может понадобится build-essential для компиляции и другие пакеты для автосборки, например autoreconf, autoconf
если у нас есть все необходимые пакеты, то мы запускаем
./autogen.sh
./configure
make
sudo make install
В таком случае по установится из текущего каталога и удалять его нужно тоже из этого каталога make uninstall

Можно добавлять/удалять/изменять различные репозитории в /etc/apt/sources.list или sources.list.d
Главное при изменениях в репозиториях не забывать обновлять индексы пакетов через
apt update

Так же сегодня начал проходить процессы. 
У всех запущенных команд, приложений и т.д есть свои процессы
Их можно просматривать например через
ps, ps aux, pstree - дерево процессов
Очень удобна утилита top для просмотра процессов в реальном времени и их нагрузки, как диспетчер задач в винде
У всех процессов есть PID - идентификатор процесса(это кстати файл, все в линукс файл)))
у процессов есть статусы
running
stopped
sleeping
zombie

Так же у процессов есть PPID это parent id, родительский процесс который породил остальные
UID - идентификатор пользователя
GID - идентификатор группы пользователя

Процессы можно убивать по PID
kill pid
либо через ctrl+c
