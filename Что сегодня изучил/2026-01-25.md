Сегодня изучал архивацию и сжатие файлов в linux
Утилита tar(файлы .tar) помогает собрать несколько файлов в один пакет .tar, который затем можно архивировать с помощью gzip,bzip2,xz
### Флаги tar
- c - создать tar пакет
- v - показать процесс, т.е. что именно архивируется
- f - указать файл (название .tar файла)
-  z,j,J - сразу же создать архив gzip,bzip2,xz соответственно
-  x - разархивировать
-  t - показать содержимое архива с путями

### Примеры команд tar
- tar -cvf filename.tar filename - создание .tar пакета с указанным файлом(можно dir/ для сбора всего каталога)
- tar -czvf arch.tar.gz dir/ - создание gzip архива из каталога
- tar -tf file.tar.gz - просмотр содержимого архива
### Некоторые команды для утилит gzip,bzip2,xz
- gzip -1 filename - собрать архив с быстрым сжатием (-9 самое сильное но долгое сжатие)
- xz --memlimit-compress=100MiB filename -k (ограничение использования ram, т.к. xz наиболее требовательная утилита, флаг -k для сохранения исходного файла иначе при сборе архива исходный файл пропадет)
- gunzip,bunzip,unxz - распаковка архивов для gzip,bzip2,xz соответственно

### Архивация zip
Zip может помочь когда нужно перенести архив на другую ос - win, macos, т.к. там они тоже прекрасно поддерживаются. Так же zip позволяет упаковывать сразу каталоги без создания пакета файлов (.tar)
- zip -r archive.zip dir/ - рекурсивно собирает архив из каталога dir
- unzip archive.zip dir - распаковка архива
- unzip -l archive.zip - посмотреть что внутри архива
- unzip archive.zip dir/config.conf - распаковать только один файл из архива(для этого нужно знать точный путь)
