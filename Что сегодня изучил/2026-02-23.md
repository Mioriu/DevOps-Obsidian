Задание:
### Практика: Настройка SSH для безопасного удаленного доступа к серверу

После изучения теоретических основ SSH, давайте закрепим полученные знания с помощью практического задания. В этой практике вы научитесь настраивать безопасное SSH-соединение с использованием ключей и конфигурационных файлов.

#### Задание

Настройте SSH для безопасного удаленного доступа к серверу.

1. Установите SSH-клиент и сервер (если не установлены)
2. Сгенерируйте пару SSH-ключей:
    - Используйте алгоритм RSA или Ed25519
    - Укажите размер ключа не менее 4096 бит для RSA
    - Добавьте комментарий, содержащий ваш email или имя устройства
3. Скопируйте публичный ключ на удаленный сервер
4. Настройте конфигурационный файл клиента (~/.ssh/config) для упрощения подключений:
    - Создайте запись для удаленного сервера с псевдонимом
    - Укажите имя пользователя, адрес сервера и путь к приватному ключу
    - Добавьте параметр ServerAliveInterval для поддержания соединения
5. Подключитесь к удаленному серверу с использованием созданного псевдонима

**Выполните все шаги задания и введите в поле ответа список использованных команд.**

**Примечание:** Если у вас нет доступа к удаленному серверу, вы можете выполнить практику на локальной машине, используя localhost в качестве сервера. В этом случае вы будете подключаться к своей же машине по SSH.

**Подсказка:** Для большей безопасности вы можете также настроить сервер SSH, отключив аутентификацию по паролю и вход под пользователем root. Однако делайте это только после того, как убедитесь, что аутентификация по ключу работает корректно.


Решение:
sudo apt update && sudo apt install openssh-server -y && sudo apt install openssh-client -y
ssh-keygen -t ed25519 -C "Matthew"
ssh-copy-id -i test_key.pub testuser@ssh_server
cat config
Host ssh_con_test
  HostName ssh_server
  User testuser
  Port 22
  IdentityFile ~/.ssh/test_key

Host *
    ServerAliveInterval 60
    ConnectTimeout 30
ssh testuser@ssh_server или ssh ssh_con_test, если прописано в клиентском конфиге

В качестве клиента-сервера развернул 2 докер контейнера в одной bridge сети, ubuntu:latest, ssh_server и ssh_client, в которых установил openssh-server/client, на ssh_server создал юзера test_user, и немного правил конфиг /etc/ssh/sshd_config
В итоге получилось корректно скопировать ключ на сервер и подключиться по ssh
