Сегодня проходил переменные окружения:
Переменные окружения необходимы для быстрого доступа к исполняемым программам, самим этим программам до конфигов утилит/других программ и т.д., для исключения указания полного пути до файлов и в целом для хранения каких либо параметров.
Пример:
у нас есть переменная PATH, которая содержит в себе /usr/bin:/usr/sbin, система будет искать исполняемые файлы сначала в путях указанных в PATH, у нас есть переменная USER, которая содержит текущее имя пользователя, переменная HOME, содержащая /home/user и т.д.
Эти переменные могут использоваться например в скриптах и программах для быстрого доступа к файлам или использования каких либо параметров.
Вывести все переменные окружения можно через printenv или env.
Вывести конкретную переменную через: echo $PATH (\$ для указания того что то именно переменная).
Переменные окружения можно записывать в файл .bashrc, который либо индивидуальный для пользователя по пути /home/username/.bashrc либо для всего сервера /etc/bash.bashrc.
Этот файл будет применяться при каждом подключении пользователя к терминалу, либо его можно применить вручную через:
source .bashrc
Создание переменной:
MY_VAR="VAR"
Для того чтобы этой переменной могли пользоваться другие программы, а не только пользователь необходимо сделать export, пример:
export MY_VAR="VAR" 
Для удаление переменной окружения используем:
unset MY_VAR

Так же сегодня изучал кастомизацию терминала, мы можем задавать различные пользовательские настройки, прописывая их в .bashrc
Например можем устанавилвать алиасы на команды alias c="clear", теперь команду clear можно будет вызывать по "c"
Можно настраивать промпт(это входящее сообщение которое мы видим перед вводом каждой команды, например user/host dir$) через переменную PS1. В общем много всего можно кастомизировать, но это со временем, пока в процессе обучения не столь важно.[[Настройка окружения командной строки]]
