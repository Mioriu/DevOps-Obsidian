
# Часть 1: Зачем нужна ротация логов

Представьте, что у вас есть ежедневник, в который вы записываете все события. Через год он станет таким толстым, что не поместится на полке. А через пять лет? Именно эту проблему **решает ротация логов**.

## Что случится без ротации?

Реальная история: на сервере забыли настроить ротацию логов веб-сервера. Через полгода:

- Файл access.log вырос до 50 ГБ
- Диск заполнился на 100%
- Сайт перестал работать (некуда писать данные)
- Даже открыть такой файл стало проблемой

## Что делает logrotate?

logrotate — это как архивариус, который:

1. **Переименовывает** старый лог (например, access.log → access.log.1)
2. **Создаёт** новый пустой файл для записи
3. **Сжимает** старые логи для экономии места
4. **Удаляет** совсем старые архивы

И всё это происходит автоматически, обычно ночью, когда нагрузка минимальна.

# Часть 2: Как работает logrotate

**`Logrotate`** обычно запускается ежедневно через **`cron`** или **`systemd timer`**. Давайте проверим:

```bash
ls -la /etc/cron.daily/logrotate
```

Или для **`systemd`**:

```lua
systemctl status logrotate.timer
```

## Основная конфигурация

**Главный конфигурационный файл**:

```bash
cat /etc/logrotate.conf
```

**Типичное содержимое**:

```1c
# Ротация еженедельно
weekly

# Хранить 4 старых версии
rotate 4

# Создавать новый пустой файл после ротации
create

# Сжимать старые логи
compress

# Включить конфигурации из директории
include /etc/logrotate.d
```

## Параметры ротации

Рассмотрим **все основные директивы** logrotate:

**Частота ротации:**

- `daily` — ротация каждый день
- `weekly` — ротация раз в неделю
- `monthly` — ротация раз в месяц
- `yearly` — ротация раз в год
- `size 100M` — ротация при достижении размера (k, M, G)

**Управление файлами:**

- `rotate N` — количество старых версий для хранения
- `maxage 30` — удалять логи старше N дней
- `missingok` — не выдавать ошибку, если файл отсутствует
- `notifempty` — не ротировать пустые файлы
- `create mode owner group` — создать новый файл с указанными правами
- `nocreate` — не создавать новый лог-файл
- `copytruncate` — копировать лог и обрезать оригинал (для программ, которые не могут переоткрыть файл)

**Сжатие:**

- `compress` — сжимать старые логи gzip'ом
- `nocompress` — не сжимать
- `delaycompress` — сжимать при следующей ротации (полезно, если процесс ещё может писать в старый файл)
- `compresscmd /bin/bzip2` — команда для сжатия (по умолчанию gzip)
- `compressext .bz2` — расширение сжатого файла
- `compressoptions -9` — опции для команды сжатия

**Размеры и условия:**

- `minsize 1M` — ротировать только если размер больше указанного
- `maxsize 100M` — принудительная ротация при превышении размера
- `ifempty` — ротировать даже пустые файлы (противоположность notifempty)

**Дата в имени файла:**

- `dateext` — добавить дату к имени ротированного файла
- `dateformat -%Y%m%d` — формат даты (по умолчанию -%Y%m%d)
- `dateyesterday` — использовать вчерашнюю дату

**Скрипты и действия:**

- `sharedscripts` — выполнить postrotate только один раз для всех файлов (а не для каждого)
- `prerotate ... endscript` — скрипт перед ротацией
- `postrotate ... endscript` — скрипт после ротации
- `firstaction ... endscript` — выполнить перед всеми ротациями
- `lastaction ... endscript` — выполнить после всех ротаций
- `preremove ... endscript` — выполнить перед удалением старого лога

**Специальные параметры:**

- `mail address` — отправить ротированный лог на email перед удалением
- `mailfirst` — отправить лог сразу после ротации
- `maillast` — отправить лог перед удалением (по умолчанию)
- `su user group` — выполнять ротацию от имени указанного пользователя
- `olddir directory` — переместить старые логи в указанную директорию

## **Пример с использованием различных директив:**

```1c
/var/log/myapp/*.log {
    daily                    # Ежедневная ротация
    rotate 7                 # Хранить 7 версий
    maxage 30               # Удалять файлы старше 30 дней
    size 100M               # Или при достижении 100MB
    minsize 1M              # Но только если больше 1MB
    compress                # Сжимать
    delaycompress           # Сжимать на следующий день
    compresscmd /bin/bzip2  # Использовать bzip2
    compressext .bz2        # Расширение .bz2
    missingok               # Игнорировать отсутствие файла
    notifempty              # Не ротировать пустые
    create 0640 app app     # Создать новый с правами
    sharedscripts           # Один postrotate для всех файлов
    dateext                 # Добавить дату
    dateformat -%Y%m%d-%s   # Формат даты
    olddir /var/log/myapp/old  # Старые логи в поддиректорию
    
    prerotate
        echo "Starting rotation" | logger
    endscript
    
    postrotate
        # Послать сигнал приложению переоткрыть логи
        /usr/bin/killall -SIGUSR1 myapp
    endscript
}
```

# Часть 3: Читаем конфигурацию

## Давайте разберём реальный пример — настройки для nginx:

```bash
cat /etc/logrotate.d/nginx
```

#### Вы увидите что-то подобное:

```1c
/var/log/nginx/*.log {
    daily           # ротировать каждый день
    missingok       # не паниковать, если файла нет
    rotate 14       # хранить 14 старых версий
    compress        # сжимать старые логи
    delaycompress   # сжимать со второй ротации
    notifempty      # не трогать пустые файлы
    create 640 nginx adm    # создать новый файл с правами
    sharedscripts   # выполнить скрипт один раз для всех файлов
    postrotate      # после ротации выполнить:
        if [ -f /var/run/nginx.pid ]; then
            kill -USR1 `cat /var/run/nginx.pid`
        fi
    endscript
}
```

### Что означает каждая строчка?

**Путь к файлам:** `/var/log/nginx/*.log` — все файлы с расширением .log в директории nginx

**Основные директивы:**

- `daily` — как часто ротировать (может быть weekly, monthly, yearly)
- `rotate 14` — сколько старых копий хранить (тут 14 дней)
- `compress` — сжимать gzip'ом (экономия места в 10-20 раз!)
- `delaycompress` — хитрость: сжимать не сразу, а при следующей ротации (вдруг процесс ещё пишет в старый файл)

**Зачем нужен postrotate?**

Некоторые программы открывают лог-файл и держат его открытым. **Если просто переименовать файл, программа продолжит писать в старый файл!** Команда `kill -USR1` говорит `nginx`: "_Переоткрой лог-файлы_"
