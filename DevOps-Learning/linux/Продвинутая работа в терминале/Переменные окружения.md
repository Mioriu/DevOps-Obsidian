## Что это?

Переменные окружения (**environment variables**) — это специальные параметры, которые хранят информацию о настройках операционной системы и окружения пользователя. Они влияют на поведение команд, приложений и скриптов, запускаемых в терминале.

Переменные окружения помогают программам понимать, где искать файлы и библиотеки, какие настройки использовать по умолчанию, и кто является текущим пользователем.

### Что такое переменные окружения на практике?

Каждая переменная окружения имеет имя (например, `HOME`, `PATH`) и значение (например, `/home/alice`, `/usr/local/bin:/usr/bin`). Вот пример нескольких переменных окружения:

```ruby
USER=alice
HOME=/home/alice
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin
SHELL=/bin/bash
```

### Просмотр переменных окружения

Для просмотра всех текущих переменных окружения вы можете использовать команду:

```bash
printenv
```

Эта команда выведет большой список переменных и их значений.

Чтобы посмотреть значение конкретной переменной, используйте команду `echo` и знак доллара (`$`):

```bash
echo $USER
echo $HOME
echo $PATH
```

### Создание и изменение переменных окружения

Создание переменной окружения можно выполнить простой командой присвоения, однако чтобы переменная стала доступной всем программам, запущенным из текущего терминала, нужно использовать команду `export`:

```makefile
MY_VARIABLE="Hello World"    # Просто переменная (локальная для текущей оболочки)
export MY_VARIABLE            # Теперь переменная доступна всем запущенным программам
```

Или в одну строку:

```bash
export MY_VARIABLE="Hello World"
```

Теперь программы, которые вы запускаете, смогут видеть и использовать `MY_VARIABLE`.

### Использование переменных окружения в командах

Переменные окружения удобны, когда вы хотите часто использовать какое-то значение или путь. Например, вы можете задать путь к своему проекту один раз, и использовать его многократно:

```bash
export PROJECT_DIR="/home/alice/myproject"

# Теперь можете использовать $PROJECT_DIR вместо длинного пути:
cd $PROJECT_DIR
ls -la $PROJECT_DIR/logs
vim $PROJECT_DIR/config.yml
```

### Удаление переменных окружения

Если переменная окружения больше не нужна, её можно легко удалить:

```bash
unset MY_VARIABLE
```

Теперь `MY_VARIABLE` удалена из окружения и больше не доступна.

### Самые важные переменные окружения в Linux

Рассмотрим наиболее распространённые переменные, с которыми вы будете сталкиваться:

- **PATH** — определяет список директорий, где система ищет исполняемые файлы (команды и приложения):
    
    ```bash
    echo $PATH
    /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin
    ```
    
    Если путь не указан в переменной `PATH`, команда не будет найдена, пока вы явно не укажете полный путь.
- **HOME** — домашний каталог пользователя:
    
    ```bash
    echo $HOME
    /home/alice
    ```
    
    Многие команды и приложения используют эту переменную, чтобы хранить настройки, кеш или временные файлы.
- **USER** — имя текущего пользователя:
    
    ```bash
    echo $USER
    alice
    ```
    
    Используется приложениями и скриптами для определения текущего пользователя.
- **SHELL** — командный интерпретатор (оболочка), используемый по умолчанию:
    
    ```bash
    echo $SHELL
    /bin/bash
    ```
    
- **EDITOR** — редактор по умолчанию, например, для редактирования файлов конфигураций:
    
    ```bash
    export EDITOR=nano
    ```
    
- **LANG** — языковые настройки окружения:
    
    ```bash
    echo $LANG
    ru_RU.UTF-8
    ```
    

### Как сделать переменные постоянными

Переменные окружения, заданные через терминал, действуют до закрытия терминала. Чтобы переменная сохранялась всегда, её нужно указать в файле настроек шелла пользователя, например в `~/.bashrc` (для bash):

1. Откройте файл настроек:
    
    ```bash
    vim ~/.bashrc
    ```
    
2. В конец файла добавьте строку с вашей переменной:
    
    ```bash
    export MY_VARIABLE="Моё постоянное значение"
    ```
    
3. Сохраните файл и примените изменения командой:
    
    ```bash
    source ~/.bashrc
    ```
    
4. Теперь переменная `MY_VARIABLE` доступна всегда, когда вы открываете новый терминал.

### Примеры использования переменных окружения в реальных ситуациях

- **Указание редактора по умолчанию для Git:**
    
    ```bash
    export EDITOR=vim
    git commit   # откроет vim для ввода сообщения коммита
    ```
    
- **Создание удобных сокращений для длинных путей:**
    
    ```bash
    export LOG_DIR="/var/log/nginx"
    tail -f $LOG_DIR/error.log
    ```
    
- **Добавление своей директории в `PATH`:**
    
    ```bash
    export PATH="$PATH:/home/alice/scripts"
    # теперь можно запускать скрипты из /home/alice/scripts откуда угодно
    ```
